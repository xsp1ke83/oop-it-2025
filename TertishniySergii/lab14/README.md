## Мета роботи 
Познайомитися з сучасними принципами проєктування програмного забезпечення та детально розглянути один із принципів SOLID.
## Хід роботи
1. Було опрацьовано теоретичний матеріал щодо принципів проєктування програмного забезпечення та визначено їх ключові характеристики. Зокрема, принципи проєктування допомагають створювати гнучкі, масштабовані та підтримувані програмні системи.
2. Особливу увагу приділено набору принципів SOLID, які описують п’ять основних правил для побудови якісних об’єктно-орієнтованих систем: SRP, OCP, LSP, ISP, DIP.
3. Було розроблено приклад програми на Python, у якому продемонстровано роботу принципу Single Responsibility Principle (SRP).
У початковому [прикладі](lab14(1).py) один клас виконував одразу дві функції - описував модель користувача та зберігав її у файл, що порушує SRP.
У виправленому [варіанті](lab14(2).py) логіку було розділено між класами User та UserSaver, кожен з яких отримав лише одну відповідальність.
4. Результати лабораторної роботи опублікував у хмарному репозиторії [GitHub](https://github.com/xsp1ke83/oop-it-2025/tree/master/TertishniySergii).

## Висновки: 
У ході лабораторної роботи було розглянуто загальні принципи проєктування програмного забезпечення та детально проаналізовано один із принципів SOLID — Single Responsibility Principle. На простому прикладі продемонстровано, як розділення обов’язків між класами дозволяє уникати зайвої залежності, спростити підтримку коду та покращити архітектуру програмної системи.
