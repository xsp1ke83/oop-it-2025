Познайомитися з групою структурних шаблонів проєктування.
## Хід роботи
1. Було опрацьовано теоретичний матеріал щодо структурних шаблонів та визначено їх ключові особливості. Зокрема, структурні шаблони описують способи побудови гнучких архітектур шляхом комбінування об’єктів різних класів.
2. Було розроблено приклад [програми](lab12.py), у якій продемонстровано роботу шаблону Adapter.
У прикладі існує сторонній клас LegacyPrinter, який має несумісний інтерфейс. Щоб використовувати його в клієнтському коді, я створив клас PrinterAdapter, що реалізує потрібний інтерфейс IPrinter і адаптує виклики до LegacyPrinter.
3. Код з реалізацією:
```python
# Старий принтер з несумісним інтерфейсом
class OldPrinter:
    def old_print(self, text):
        return f"Old printer prints: {text}"

# Новий інтерфейс, який очікує клієнт
class PrinterInterface:
    def print(self, text):
        raise NotImplementedError

# Адаптер: робить OldPrinter сумісним з новим інтерфейсом
class PrinterAdapter(PrinterInterface):
    def __init__(self, old_printer):
        self.old_printer = old_printer

    def print(self, text):
        return self.old_printer.old_print(text)

old_printer = OldPrinter()
adapter = PrinterAdapter(old_printer)

print(adapter.print("Hello world!"))
```
