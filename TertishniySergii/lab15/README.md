## Мета роботи 
Познайомитися з основними принципами, техніками та поняттям “запахів коду” в процесі рефакторингу програмного забезпечення.
## Хід роботи
1. Було опрацьовано теоретичний матеріал щодо рефакторингу програмного забезпечення, його мети та ключових принципів.
Рефакторинг спрямований на покращення внутрішньої структури коду без зміни зовнішньої поведінки програми. Основні принципи, які було розглянуто: маленькі кроки, часті перевірки, наявність тестів, збереження функціоналу та підвищення зрозумілості коду.
2. Було опрацьовано найбільш поширені техніки рефакторингу. Для детального розгляду було обрано техніку Extract Method, яка дозволяє виділити частину коду в окремий метод для підвищення читабельності й зменшення дублювання.
3. Було реалізовано [приклад](lab15(1).py) до техніки Extract Method.
